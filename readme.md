# 📈 EasyQuant (简易量化)

> **基于 LangGraph 的双轨多智能体量化投研终端**

EasyQuant 是一个现代化的量化投研分析系统。本项目采用前沿的 Multi-Agent（多智能体）架构，彻底解决了通用大模型在金融垂直领域经常出现的“数据幻觉”痛点。通过底层的意图调度网关，系统能够动态挂载真实的 Tushare 金融数据与本地知识库，为用户提供集实时盯盘、热点追踪、深度研报与自然语言交互于一体的极客级终端体验。

## ✨ 核心特性

* **🧠 双轨智能体架构 (Dual-Track Agent System)**：系统在底层物理隔离了“实时问答”与“长文诊断”两条链路，确保不同复杂度的任务由最匹配的 Agent 图谱处理。
* **🛡️ 真实数据防幻觉 (Fact-Grounded & No Hallucination)**：全面接入 Tushare Pro 实时数据与本地 MongoDB 数据库，大模型（LLM）回答严格受限于 `@tool` 注入的客观数据，拒绝盲目预测与脑补。
* **🛠️ 7 大核心实战工具箱 (Tool Calling)**：智能体原生掌握 7 项核心量化分析工具，支持自然语言无缝唤醒：均线测算、高低点回撤、量能异动、大盘热点、核心估值、近期走势、基本面查询。
* **⚡ 高性能异步全栈 (Async Full-Stack)**：后端基于 FastAPI 与 Asyncio 构建，数据库交互 (`Motor`) 与 API 请求 (`HTTPX`) 全链路异步化；配合 `APScheduler` 定时任务，静默维持底层数据的每日鲜活。
* **💻 极客风无感交互 (Geek UI)**：前端抛弃沉重的框架，采用纯原生 JavaScript 与 CSS3 构建暗黑极客主题，支持流畅的动态数据渲染与微动画补间。

## 🕵️‍♂️ 多智能体角色编排 (Multi-Agent Workflow)

本系统的核心壁垒在于基于 LangGraph 状态机（StateGraph）构建的虚拟投研团队。当用户下达指令时，系统内部将触发以下角色的自动化协同：

1.  **🚦 意图调度网关 (Dispatcher)**
    * **角色定位**：系统的“前台大堂经理”。
    * **核心职能**：拦截用户的自然语言输入，进行精准的语义路由。如果用户问“大盘怎样”，它会将任务轻量级地抛给「问答 Agent」；如果用户点击“诊断”或要求“分析某只股票”，它会精准提取 `ts_code` 并将任务重负载地派发给「研报工作流」。
2.  **🔍 数据调研员 (Researcher)**
    * **角色定位**：不知疲倦的“资料搜集专员”。
    * **核心职能**：在研报生成阶段被唤醒，负责调用底层工具箱，前往 MongoDB 和 Tushare 接口中捞取目标股票的各类切面数据（基本面、资金面、技术面指标），为后续分析提供纯粹的“客观事实”。
3.  **✍️ 首席分析师 (Analyst / Writer)**
    * **角色定位**：具备金融逻辑的“投研主笔”。
    * **核心职能**：接收调研员传回的杂乱数据碎片，利用大模型强大的上下文理解能力，将这些冰冷的数据拼接成逻辑严密、结构清晰的深度诊断报告，并辅以自然语言的投资视角解读。
4.  **🧐 风控审查员 (Supervisor / Reviewer)**
    * **角色定位**：把控质量的“冷酷质检员”（引入反思机制 Reflection）。
    * **核心职能**：负责对分析师生成的“研报初稿”进行 Review。它会检查报告字数是否达标、是否存在数据缺失或逻辑硬伤。一旦发现问题，它会生成一段 `critique`（批评指导），并打回要求重写（触发 `retry_count` 循环），直到研报质量达标才允许输出给前端用户。

## 📦 技术栈 (Tech Stack)

* **AI 编排框架**: LangChain, LangGraph
* **底层大语言模型**: 兼容 OpenAI 接口规范的 LLM (默认配置为 Qwen3-VL)
* **后端 Web 框架**: FastAPI, Uvicorn (ASGI)
* **数据源与处理**: Tushare Pro, Pandas
* **持久化与存储**: MongoDB, Motor (Async Driver)
* **本地向量检索**: Sentence-Transformers (为后续 RAG 预留)
* **任务调度与网络**: APScheduler, HTTPX

## 🚀 快速开始

### 1. 环境准备
推荐使用 Python 3.10+ 环境。克隆项目后，安装核心依赖包：

```bash
pip install -r requirements.txt
```

### 2. 配置环境变量

在项目根目录创建 `.env` 文件，填入你的专属密钥与配置：

```env
# 大模型配置
LLM_API_KEY=your_api_key_here
LLM_BASE_URL=https://api.your-llm-provider.com/v1
LLM_MODEL=qwen3-vl-plus-2025-09-23

# Tushare 金融数据 Token (必填)
TUSHARE_TOKEN=your_tushare_token_here

# MongoDB 连接 URI
MONGO_URI=mongodb://127.0.0.1:27017

# HuggingFace 国内镜像 (解决向量模型下载超时问题)
HF_ENDPOINT=https://hf-mirror.com
```

### 3. 启动服务

确保本地 MongoDB 数据库已启动，然后在终端运行启动命令：

```bash
python main.py
```

当终端打印出 `🚀 简易量化 (EasyQuant) 终端已启动！` 后，直接在浏览器中双击打开项目目录下的 `pc_terminal.html` 即可开始体验。

## 💡 对话范例 (Prompt Examples)

在终端右侧的智能聊天框中，你可以直接输入以下自然语言向 Agent 提问：

- **趋势发问**："科大讯飞现在破 20 日线了吗？"
- **位置发问**："工商银行现在在底部吗？距离今年最高点跌了多少？"
- **量能发问**："今天大盘热点是什么？哪些板块有资金领涨？今天大盘放量了吗？"
- **估值发问**："帮我看看这只股票现在的估值贵不贵？盘子有多大？"
- **综合分析**："总结一下这只股票近期的走势，并结合它的市盈率给我一个基本面诊断。"
